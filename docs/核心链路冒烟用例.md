# 核心链路冒烟用例

目标：每次改动后用 5-10 分钟快速验证核心能力是否可用。

## 覆盖项

- 账号登录
- 短信发送 + 短信登录
- 文件上传 -> 持久化 -> 对话引用
- Chat SSE 首 token 返回
- 会话历史查询

## 脚本位置

- `scripts/core-smoke.mjs`

## 运行前准备

1. 启动后端服务（默认 `http://127.0.0.1:8080`）。
2. 准备账号登录参数（账号 + 验证码 + uuid）。
3. 准备短信登录参数（手机号 + 验证码）。

说明：  
- 若不传完整参数，对应登录项会失败并打印原因。  
- 脚本只用系统自带 `node`，不新增依赖。

## 环境变量

- `SMOKE_BASE_URL`：后端地址，默认 `http://127.0.0.1:8080`
- `SMOKE_ACCOUNT_USERNAME`：账号用户名
- `SMOKE_ACCOUNT_PASSWORD`：账号密码
- `SMOKE_ACCOUNT_CODE`：图形验证码
- `SMOKE_ACCOUNT_UUID`：验证码 uuid
- `SMOKE_SMS_PHONE`：短信登录手机号
- `SMOKE_SMS_CODE`：短信验证码

## 运行命令（PowerShell）

```powershell
$env:SMOKE_BASE_URL="http://127.0.0.1:8080"
$env:SMOKE_ACCOUNT_USERNAME="admin"
$env:SMOKE_ACCOUNT_PASSWORD="admin123"
$env:SMOKE_ACCOUNT_CODE="xxxx"
$env:SMOKE_ACCOUNT_UUID="xxxx-xxxx-xxxx"
$env:SMOKE_SMS_PHONE="13800000000"
$env:SMOKE_SMS_CODE="123456"
node scripts/core-smoke.mjs
```

## 结果判定

- 任一项失败：脚本退出码 `1`。
- 全部通过：脚本退出码 `0`，输出“全部通过”。
