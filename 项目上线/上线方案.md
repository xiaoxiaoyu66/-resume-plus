● 一、最低成本部署方案

  你的项目现在依赖：MySQL + Redis + MinIO + Elasticsearch + PostgreSQL(PGVector) + RabbitMQ，全跑起来太沉了。精简方案：

  方案 A：单机部署（推荐，约 50元/月）

  买一台 2核4G 阿里云/腾讯云学生机（约 50元/月），改造架构：

  [1台 VPS :8080]
  ├── Spring Boot JAR (后端)
  ├── Nginx (前端静态文件 + API 反向代理)
  ├── MySQL (用 LightDB 或直接 MySQL)
  ├── Redis (可选，可以先用本地 Map 替代缓存)
  └── 文件存储 (本地磁盘，不用 MinIO)

  去掉的服务：
  - RabbitMQ → 改成同步处理或线程池异步，文件上传不多的话影响不大
  - Elasticsearch → 数据量小时直接用 MySQL LIKE 查询，或者用 PGVector 替代
  - MinIO → 文件存本地，Nginx 直接 serve
  - PGVector → 如果非要用向量检索，可以在同一个 MySQL 里存向量（MySQL 8.0 也支持），或者直接调用 SiliconFlow API
    做检索，不需要本地向量库

  具体改法：

  # 简化后的 application.yml
  minio:    # 改成本地文件
    endpoint: file:///data/uploads

  elasticsearch: # 去掉
    enabled: false

  rabbitmq: # 改成同步
    enabled: false

  redis: # 本地缓存兜底
    enabled: false
    # 依赖已有的 L1 本地缓存（你代码里已经有这个降级）

  配合 Docker Compose 启动只要 MySQL 一个依赖，压力小很多。

  方案 B：极致省钱（0~20元/月）

  - 前端 → 部署到 Vercel（免费）或 Netlify
  - 后端 → 部署到 Railway 或 Zeabur（免费额度够个人用）
  - 数据库 → 用 Supabase 免费 PostgreSQL（送 500MB，够用了）
  - 文件存储 → Cloudflare R2（免费 10GB）或直接用 Supabase Storage
  - Redis → 不需要，本地缓存扛得住

  Vercel (前端静态)
      │
  Zeabur (Spring Boot JAR)
      │
  Supabase (PostgreSQL, 含 PGVector)

  成本：零元，三个平台都有免费额度。缺点：后端冷启动慢（免费容器会休眠）。

  方案 C：直接用学生机（最省心）

  阿里云 "云翼计划" — 学生认证后 9.5元/月 2核2G，装 Docker Compose，把 MySQL 和 Redis 跑起来就行，Elasticsearch / MinIO
  去掉。

  二、必须优化的点（按优先级排序）

  P0 - 数据安全性

application.yml 里硬编码了大量密钥

  deepseek.api-key: sk-xxx   # ← 泄露了，换掉
  aliyun.sms.access-key-secret: xxx  # ← 换掉
  wx.app-secret: xxx  # ← 换掉

  改法：环境变量注入，application.yml 里只写 ${DEEPSEEK_API_KEY}。密钥全部重置。

  P0 - 删除无用依赖

  pom.xml 里 spring-boot.version 写的 4.0.3，实际 Spring Boot 官方还没发 4.x。推测是你改了版本号但实际用的是
  3.x。检查一下 mvn dependency:tree，确保版本匹配，否则迟早出兼容问题。

  P1 - 配置抽离

  application.yml 快300行了，把 Aliyun、微信、MinIO、PGVector、DeepSeek 这些拆成独立的配置文件。

  P1 - 错误处理

  AI 对话里如果 DeepSeek API 超时或返回报错，前端现在可能直接白屏或一直 loading。

  P2 - 加测试

  哪怕只写几个 @SpringBootTest 的接口冒烟测试，面试时都能拿出来说。

  P2 - Docker 化改造

  现状的前后端 Dockerfile 只用于生产构建。改成开发也能用：

  # 开发阶段用这个，省去本地配 MinIO/Redis 的麻烦
  services:
    mysql:     # 已有
    redis:     # 已有
    app:       # 加这个
      build: .
      ports: ["8080:8080"]
      depends_on: [mysql, redis]
